# Developer Outreach

The first contributor pitch should be practical:

- Lens is small enough to understand in one sitting.
- It has offline tests and no API-key barrier.
- It sits next to Pulse, so contributors can work on either fabric exports or
  interpret adapters.
- Good first issues are adapter fixtures, citation formatting, and docs.

## Maintainer Actions

1. Pin three starter issues: JSON-LD fixture, local file adapter examples, and
   Pulse snapshot contract docs.
2. Ask focused feedback from graph RAG, p2p, DID/VC, and agent tooling builders.
3. Publish the GitHub Pages docs after the repo is live.
4. Add repository topics: `knitweb`, `pulse`, `rag`, `provenance`,
   `content-addressing`, `p2p`, `pure-python`.
5. Keep the README examples runnable without credentials.
